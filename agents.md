## 概览

我希望设计一个番茄 chrome 插件。除了基础的番茄功能，例如 25 分钟专注；5 分钟休息；重置、跳过外，只解决一个问题：25分钟到了 却停不下来，通知看到了，但你对自己说"再 5 分钟就好/这段写完就停"，然后 5 分钟变 20 分钟。问题在于"看到了但不想停"。

## 功能规划

- **强制休息模式**：时间到后，扩展进入"休息锁定"——所有标签页注入一个全屏覆盖层，**无跳过按钮**。听起来狠，但这正是花时间做这个插件的目的。详见"特殊约定 / 休息不提供跳过出口"。
- **延后有代价**：允许"再 5 分钟"按钮，但点一次扣一次当日配额（每天只能加时 3 次）。把无限续杯变成有限续杯。
- **结束要说出口**：时间到时弹窗问"这一番茄你完成了什么？"——打一行字才能进入休息。两个作用：一是打断心流惯性（被迫切换到元认知层），二是把做完的事落地，心理上更容易放手。
- **反向激励**：连续 N 天按时休息，徽章 / 界面有视觉奖励（比如小图鉴、占满全屏的 love monster）。这个设计的精髓是：**奖励"停下来"，而不是奖励"做了多少番茄"**。番茄钟 App 常见的错误是把奖励绑在"完成番茄数"上，这会强化你不想停的那个倾向。

## 技术栈

- Chrome Extension **Manifest V3**
- 原生 HTML / CSS / JS，无构建工具（扩展体积小、加载快）
- 状态持久化：`chrome.storage.local`（唯一事实来源）
- 计时：`chrome.alarms` + `endTime` 时间戳（Service Worker 会被 Chrome 随时休眠，不能依赖内存变量或 `setInterval`）
- 锁屏：Shadow DOM content script，防止被页面 CSS 污染
- 音频：Web Audio API 程序生成（白噪音、叮声），通过 **offscreen document** 播放（MV3 的 SW 无 DOM，无法直接播音）
- 视觉素材：SVG（图标、锁屏背景、小怪兽主题）

## 设计与交互系统（UI 规范）

### 视觉基调

- 主色：番茄红 `#d94f3a`
- popup 宽度 280px，偏紧凑；锁屏全屏覆盖所有标签页
- 字体：系统字体栈（`-apple-system, BlinkMacSystemFont, 'PingFang SC'...`），避免加载外部字体
- 锁屏默认半透明深色 + 背景模糊；可切换为 SVG 风景图或小怪兽主题

### 锁屏背景

默认是 `rgba(18,18,18,0.88)` + `backdrop-filter: blur(8px)`；用户也可在 popup 设置里切换为预设 SVG 风景：**黎明 / 深海 / 暮色**。图片模式下用一层 `rgba(0,0,0,0.45)` 暗化蒙层保证白字可读。

### 情绪小怪兽主题

致敬 Anna Llenas 的绘本 *The Color Monster*——每只小怪兽代表一种情绪，用颜色区分。本主题借用这个隐喻，把番茄钟各阶段的用户心境映射到对应的小怪兽上。

**状态 → 小怪兽映射**：

| 场景                         | 小怪兽 | 颜色 | 心境                                              |
|------------------------------|--------|------|---------------------------------------------------|
| 点击"开始专注"之前（IDLE）    | Happy  | 黄   | 你还没开始，一切安好                              |
| 专注期间（FOCUSING）          | Calm   | 绿   | 抱胸闭眼，专注不打扰                              |
| 强制休息锁屏（BREAKING）      | Angry  | 红   | 摇晃着喊"离开电脑去休息！"                        |
| Hover 加时按钮               | Afraid | 深灰 | 从按钮下跳出，配一句"确定吗？/ you sure?"——可视化心里那个"再 5 分钟"的犹豫 |

**未做的**：Love（粉）用于反向激励——连续 N 天按时休息时占满全屏，作为"奖励停下来"的视觉反馈。

**实现要点**：

- 4 只小怪兽都是 SVG，用 `<filter>` 的 turbulence + displacement 模拟蜡笔毛绒边缘
- Afraid monster 必须挂在 `document.documentElement`（**不能在 shadow DOM 里**），否则加时成功后锁屏销毁，它的动画会被中途打断
- 挂到 document 级别时要用 `all: initial` 对抗页面 CSS 污染（常见元凶：`img { max-width: 100% }` 会把 img 压成 0 宽）
- 致敬原作，不复刻原画（版权考虑）

### 休息不提供跳过出口

原需求里提到"关闭需要明确理由，比如点三次确认"。初版做了三次确认按钮，但上线加时机制（消耗配额把状态切回专注）之后，跳过按钮变成了**不扣配额的后门**：想继续干活走加时，不想继续走跳过，两条路径对同一个欲望，其中一条没有代价。这与"强制休息"的整个设计方向矛盾。

所以现在：
- 锁屏上没有"跳过休息"按钮
- popup 在休息期间"跳过"按钮也是 disabled
- SW 的 `skip()` 在 break 阶段无效（防御性兜底）

休息期间只有两种合规出口：
1. **加时**：扣配额，回到专注
2. **等待**：让休息自然结束

真遇到紧急情况（比如要立即开会），用户必须去 `chrome://extensions` 禁用扩展——这条路径的摩擦足够大，且让用户清楚感到"我在绕过系统"，反而是更诚实的出口。

### 测试模式（TEST_MODE）

正式番茄周期是 25 分钟专注 / 5 分钟休息，手工测试一轮要至少半小时，反复调试时不现实。所以代码里内置一个 `TEST_MODE` 开关，让一轮在 45 秒内跑完。

**开关位置**（两处必须同时改）：
- [background/service-worker.js](background/service-worker.js) 顶部：`const TEST_MODE = true;`
- [content/lockscreen.js](content/lockscreen.js) 顶部：`const TEST_MODE = true;`

两个文件必须保持一致，因为 SW 决定计时器时长，锁屏决定"加时按钮"的时长折算。

**TEST_MODE = true 时的行为**：
- 专注 15 秒 / 休息 30 秒（而非 25 分钟 / 5 分钟）
- 锁屏**预设按钮**按"秒"算：`+5 分钟` 实际加 5 秒，`+25 分钟` 加 25 秒
- 锁屏**自定义输入**始终按真·分钟换算，**不受 TEST_MODE 影响**。支持小数，例如 `0.5` = 30 秒、`1.5` = 90 秒。范围 0.1–120 分钟。
- 每次"重新加载扩展"会自动清零当日配额（`chrome.runtime.onInstalled` 里的逻辑），方便反复测试无须手动重置
